## 思考题

- 你理解的对于类似ucore这样需要进程/虚存/文件系统的操作系统，在硬件设计上至少需要有哪些直接的支持？至少应该提供哪些功能的特权指令？

  硬件支持：串口控制器、存储控制器、ROM、内存、TLB、数据旁路、运算器、寄存器等

  特权指令：各类异常处理指令、中断指令、涉及TLB的操作指令等  

- 你理解的x86的实模式和保护模式有什么区别？你认为从实模式切换到保护模式需要注意那些方面？

  x86的实模式下只允许使用20位的地址总线，总共寻址1MB的大小，没有提供段保护；保护模式下全部32位均有效，总共寻址4GB的大小，提供段保护。从实模式切换到保护模式时，主要要注意建立全局描述符号表，并且注意指针的选择子位置；从保护模式切换到实模式时，主要要注意不能从32位代码段返回实模式，而要从16位返回，还要使得对应段描述符高速缓冲寄存器中含有合适的段界限和属性值。

- 物理地址、线性地址、逻辑地址的含义分别是什么？它们之间有什么联系？

  物理地址：即CPU在硬件平台上用于访问存储设备的实际最终地址，一般通过总线传递。

  线性地址：逻辑地址到物理地址变换之间的中间层，处理器通过段机制控制形成地址空间。

  逻辑地址：即代码中给出的地址，指令之中给出的地址。

- 你理解的risc-v的特权模式有什么区别？不同 模式在地址访问方面有何特征？

  risc-v定义了四种特权级模式。级别0为用户/应用程序，级别1为管理员，级别2为Hypervisor，级别3为机器。不同模式通过地址的特殊位和偏移段来标明该地址的使用标准和可访问性。具体来说，使用标准分为标准和非标准，可访问性分为读/写、只读和读/写 阴影。

- 理解ucore中list_entry双向链表数据结构及其4个基本操作函数和ucore中一些基于它的代码实现（此题不用填写内容）  

- 对于如下的代码段，请说明":"后面的数字是什么含义
```
 /* Gate descriptors for interrupts and traps */
 struct gatedesc {
    unsigned gd_off_15_0 : 16;        // low 16 bits of offset in segment
    unsigned gd_ss : 16;            // segment selector
    unsigned gd_args : 5;            // # args, 0 for interrupt/trap gates
    unsigned gd_rsv1 : 3;            // reserved(should be zero I guess)
    unsigned gd_type : 4;            // type(STS_{TG,IG32,TG32})
    unsigned gd_s : 1;                // must be 0 (system)
    unsigned gd_dpl : 2;            // descriptor(meaning new) privilege level
    unsigned gd_p : 1;                // Present
    unsigned gd_off_31_16 : 16;        // high bits of offset in segment
 };
```

每一个域在结构体中所占的位数，表示这个成员变量占多少位。

- 对于如下的代码段，

```
#define SETGATE(gate, istrap, sel, off, dpl) {            \
    (gate).gd_off_15_0 = (uint32_t)(off) & 0xffff;        \
    (gate).gd_ss = (sel);                                \
    (gate).gd_args = 0;                                    \
    (gate).gd_rsv1 = 0;                                    \
    (gate).gd_type = (istrap) ? STS_TG32 : STS_IG32;    \
    (gate).gd_s = 0;                                    \
    (gate).gd_dpl = (dpl);                                \
    (gate).gd_p = 1;                                    \
    (gate).gd_off_31_16 = (uint32_t)(off) >> 16;        \
}
```
如果在其他代码段中有如下语句，
```
unsigned intr;
intr=8;
SETGATE(intr, 1,2,3,0);
```
请问执行上述指令后， intr的值是多少？

0x10002
